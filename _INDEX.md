# warrenville-proud

Source for **warrenvilleproud.com**: a free, community-run local hub for Warrenville, Illinois (events calendar + twice-weekly blog), with **Sound & Fury Print Shop** as founding sponsor and a rotating Local Business Spotlight sellable to other businesses. Self-sustaining model: useful free content draws local readers, local sponsors fund it. Created **2026-06-17**. SEO/GEO optimized **2026-06-17**.

Static site generated by a Python script from JSON data. Refreshed automatically by two weekly scheduled tasks. Has its own `CLAUDE.md`. Launch steps in `PUBLISH.md`.

## SEO / GEO summary

Per-page titles + meta descriptions, canonical URLs, Open Graph + Twitter cards, local geo meta, and a branded preview image. Structured data: Organization + WebSite site-wide, Event ItemList on home/events, BlogPosting + BreadcrumbList on posts, FAQPage + AboutPage on About (Sound & Fury represented as sponsor entity throughout). GEO: `llms.txt`, AI-crawler allowlist in `robots.txt`, an FAQ answering natural-language local queries, and clean static HTML that AI engines can read. 24 JSON-LD blocks, all validated.

## Known issue

The in-sandbox `.git` directory has stale lock files that Cowork cannot remove (the mount blocks `unlink`, but `rename` works, so each run renames any `.git/*.lock` out of the way before committing). Pushing from the sandbox also fails because it has no GitHub credentials. Working publish path: commit in the sandbox, then run `git push origin main` on the Mac (for example via osascript `do shell script`), which uses the Mac keychain credentials. That is the standard end-of-run publish step.
